The first beta operational release of the risk threshold database is the initial version of the database that is made available for use by a select group of stakeholders for testing and feedback before it is released to the wider audience. This version details a methodology for developing and setting the risk thresholds. The beta release is intended to provide an opportunity for stakeholders to test and provide feedback on the database, and to help identify any issues or areas for improvement before the database is fully launched.

The platform allows for basic functionalities such as adding/deleting existing records from the database based on nace codes or asset tags and include information on Likelihood of Disruption and its severity. To ensure consistency and comparability across different infrastructure projects, it's important to use a consistent methodology for setting risk thresholds. We have developed a standardized framework for assessing climate change risks and setting risk thresholds based on fragility curves that you see on the right side. Climate change risks are constantly evolving, so it's important to update your risk threshold database regularly to reflect new data and insights. This can help ensure that your risk thresholds remain relevant and effective over time. This version of the database includes UI/UX and backend capabilities to execute these functionalities and provide a collaborative playform for crowdsourcing risk thresholds for various asset types.

RTD Release notes

Versioning is a process of tracking changes to a database or software over time, allowing users to understand how the database has evolved and to easily roll back changes if necessary. As changes are made to the UI/UX of the database, it will be documented here in the release notes and this will include a brief description of the change, and the date it was made. We also use GitHub as the version control system to manage changes to the database by our developers. This allows us to track changes over time, roll back changes if necessary, and collaborate with others on the database.
